Jupyter로 개발부터 블로깅까지¶

빅데이터상에서 데이터를 다루고 분석하다보니 여러 시스템과 여러 언어를 시험삼아 사용해야 하는 상황에서 하나의 툴로 최대한 많은 영역을 커버하는 편리한 개발 환경을 구축하기 어렵습니다.

Jupyter는 데이터 과학자에게는 대단히 유용한 개발툴임에 틀림없습니다. 여러 툴들을 전전했지만, 이글을 쓰는 시점에 개인 노트북안에서 Jupyter 만큼 여러 분석툴과 문서화까지 다양하게 응용하여 사용 가능한 툴을 찾지 못했습니다.

저와 비슷한 상황이신 분들에게 도움이되었으면 좋겠네요.. ㅋ

환경¶

  • 노트북: MacBook Pro (Retina, 15-inch, Mid 2014)
  • 운영체제: OSX Elcapitan 10.11.6 (15G31)
  • 개발언어: Python, Scala, Lua, Julia, R
  • 개발환경: Apache Spark, 그외 언어 종속적 Machine Learning Library 들.. (Tensorflow, Torch, skit-learn, ...)
  • 블로깅: Github Pages, Pelican

Home Brew¶

OSX 에서 linux 개발 툴을 사용하기 위해 필수적으로 필요한 패키지 관리자입니다.

http://brew.sh/index_ko.html

설치는 간단합니다. 아래와 같이 스크립트를 수행해 주면 됩니다.

> /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

이제 필요한 linux 개발 툴들이 있으면 brew install 명령을 통해 손쉽게 설치할수 있습니다.

Python & Jupyter¶

brew를 이용하여 python 및 jupyter 설치합니다. python 버전은 2.x 를 사용하는 것이 공통으로 사용할때 문제를 덜 일으키기에 2.x 를 사용합니다. 여러 버전의 python 을 사용하시는 분들은 virtualenv 패키지를 통해 분리된 환경내에서 python을 사용하시는 것을 권장합니다.

python를 위한 패키지 관리자인 pip를 설치하고 pip를 이용하여 jupyter를 설치합니다.

jupyter의 의존 패키지인 zeromq를 brew로 설치합니다.

> brew install python
> sudo easy_install pip
> pip install jupyter
> brew install zmq

jupyter 명령을 통해 jupyter notebook 에 접속합니다.

> jupyter notebook

이제 jupyter를 통해 python 을 사용할수 있습니다. 또한 jupyter에서는 terminal 기능도 제공하므로 jupyter를 띄울때 외에는 jupyter를 벗어나지 않고 거의 모든 작업이 가능합니다.

jupyter에서 terminal 창을 열고 python 으로 이미지를 로드하기 위해 pip 를 통해 Image 패키지를 설치합니다.

> pip install Image

그리고 그 화면을 Cmd + Shift + 4 키로 스크린 샷을 떠서 저장한 뒤에 블로그를 작성중인 이 notebook 에서 이미지를 로드해 뿌려보겠습니다.

이처럼 jupyter를 블로깅에 이용하면 따로 개발 환경을 띄워서 작업할 필요없이 블로그 작성 중간중간 코드를 이용하여 결과를 보면서 정리하는 것이 가능해집니다. 정말 막강한 편리함이죠.

기존 처럼 블로깅을 하려면 코드를 붙여 넣고 결과를 다시 스크린 샷으로 저장하고 블로그에 붙여넣고.. ㅠㅠ 블로깅 하려다가 포기하게 됩니다. ㅋ

여기서는 가장 기본적인 언어인 python만을 설명드렸지만, jupyter는 python 말고도 다양한 개발 언어를 지원합니다. 다른 언어들과의 연동은 다음 글에서 기회가 있으면 다루어 보도록 하겠습니다.

Jupyter, Pelican & Github Pages로 블로깅하기¶

github 하면 개발 소스 저장소가 가장 먼저 떠오르지만, github 에서는 얼마전부터 pages 라는 서비스를 통해 github 저장소에 올려놓은 웹 페이지 소스를 브라우저를 통해 볼수 있도록하면서, 웹 서버가 없어도 개인이 static 블로깅이 가능하도록 서비스하고 있습니다.

wordpress, blogger, tistory 등등 수많은 블로그 시스템이 존재하지만, 개발을 하다가 그때그때 편리하게 블로깅 가능한 면에서는 이 조합을 따라올 수 있는 블로깅 환경이 없는거 같습니다.

https://github.com/ 에 계정을 만듭니다. pelican(http://docs.getpelican.com/en/3.6.3/)으로 블로깅을 하기 위해서 2개의 repository 를 생성합니다.

  • [자신의 github id].github.io : 이 계정은 pages 를 위한 저장소입니다.
  • [자신의 github id].github.io-src : 이 계정은 pelican 을 통해 pages 에 게시될 글들의 저장하기 위한 저장소 입니다.

github 에서는 맥 사용자를 위해 Gtihub Desktop이라는 편리한 github gui 클라이언트 툴을 제공합니다. (https://desktop.github.com/)

이제 저장소를 clone 하고 pip 를 이용하여 pelican 을 설치합니다.

> git clone https://github.com/comafire/comafire.github.io
> git clone https://github.com/comafire/comafire.github.io-src
> pip install pelican markdown

이제 src 저장소로 이동하여 pelican 에 ipynb 블러그인을 설치합니다.

ipynb 플러그인은 jupyter 로 작성한 글을 html 형식으로 변환해 주는 플러그인입니다. ipynb 플러그인을 통해 변환된 html 파일이 pelican 의 기본 테마와 잘 맞지 않아 저는 clean-blog 테마를 좀 수정하여 사용하고 있습니다. 테마 설치도 같이 합니다.

> cd comafire.github.io-src
> pelican-quickstart
> git submodule add git://github.com/danielfrg/pelican-ipynb.git plugins/ipynb
> git submodule add git://github.com/gilsondev/pelican-clean-blog themes/clean-blog
> ls
LICENSE            develop_server.sh    output            plugins            workspace
Makefile        fabfile.py        pelicanconf.py        publishconf.py
content            footer            pelicanconf.pyc        themes
> ls plugins/
ipynb
> ls themes/
clean-blog

이제 ipynb 플러그인 및 테마 사용을 위하여 pelicanconf.py 설정파일을 수정해줍니다.

> vi pelicanconf.py
...
THEME = "/Users/comafire/Projects/comafire.github.io-src/themes/clean-blog"

MARKUP = ('md', 'ipynb')

PLUGIN_PATH = './plugins'
PLUGINS = ['ipynb.markup']
...

이제 jupyter 로 작성한 ipynb 노트북 파일을 meta 파일과 함께 content 디렉토리안에 위치 시킵니다.

> cd content
> ls
2016-08-26-Jupyter-From-Dev-To-Blogging.ipynb
2016-08-26-Jupyter-From-Dev-To-Blogging.ipynb-meta

meta 파일의 내용은 아래와 같이 채워 넣으면 됩니다. 글 작성시마다 파일을 하나더 생성해야 하는 번거로움은 있지만, 참을만 합니다. ㅋ

이제 작성한 글을 pelican 을 통해 html 으로 변환 시킵니다.

> pelican content
Done: Processed 1 articles, 0 drafts, 0 pages and 0 hidden pages in 0.46 seconds.
> ls output
2016-08-26-Jupyter-From-Dev-To-Blogging.html            
...

output 폴더의 내용을 github pages 를 위해 만들어 놓은 저장소에 복사해놓고 commit & push 하면 끝입니다.

> cp -R output/* ../comafire.github.io
> cd comafire.github.io/
> git add .
> git commit -m 'post blogging'
> git push
