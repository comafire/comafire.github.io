Vagrant와 Virtualbox를 이용한 개발환경¶

Vagrant는 설정파일을 통해 Virtualbox에서 Virtual Network상에 복수의 Virtual Machine을 생성/삭제할 수 있는 편리한 툴입니다. 개발을 위해 싱글머신에서 가상의 클러스터가 필요할 경우 Vagrant와 Virtualbox를 이용하여 가상의 클러스터 개발환경을 구축할 수 있습니다.

Virtualbox 설치¶

Ubuntu¶

Virtualbox 의 Repository 를 APT 소스 리스트에 추가하고 Virtualbox 최신버전을 설치합니다.

wget -q https://www.virtualbox.org/download/oracle_vbox_2016.asc -O- | sudo apt-key add -
wget -q https://www.virtualbox.org/download/oracle_vbox.asc -O- | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] http://download.virtualbox.org/virtualbox/debian $(lsb_release -cs) contrib"
sudo apt update
sudo apt install -y virtualbox virtualbox-ext-pack

MaxOS¶

https://www.virtualbox.org/ 홈페이지를 방문하여 "Downloads" 메뉴에서 "OS X hosts" 링크를 통해 dmg 파일을 다운로드 받아 설치합니다.

Vagrant 설치¶

Ubuntu¶

Vagrant 최신버전을 다운받아 설치합니다. Vagrant 의 설정파일에서 Disksize를 조정할 수 있도록 vagrant-disksize 플러그인도 설치해 줍니다.

wget -c https://releases.hashicorp.com/vagrant/2.0.3/vagrant_2.0.3_x86_64.deb -P /tmp
sudo dpkg -i /tmp/vagrant_2.0.3_x86_64.deb
vagrant plugin install vagrant-disksize

MacOS¶

https://www.vagrantup.com/ 홈페이지를 방문하여 "Downloads" 메뉴에서 "macOS" 용 dmg 파일을 다운로드 받아 설치합니다.

터미널 창을 통해서 vagrant-disksize 플러그인을 설치합니다.

vagrant plugin install vagrant-disksize

Vagrantfile¶

Vagrant는 Vagrantfile를 이용하여 다양한 가상 환경을 설정할 수 있습니다.

vagrant를 위한 디렉토리를 생성 후 초기화 명령을 수행하면 기본 Vagrantfile 이 생성되며, 이 파일을 용도에 맞게 수정하여 사용하면 됩니다.

아래는 Single-VM 과 Multi-VM 구성 예 입니다.

Single-VM 예¶

vagrant 초기화

mkdir single
cd single
vagrant init

설정파일 Vagrantfile의 예

# -*- mode: ruby -*-
# vi: set ft=ruby :

# All Vagrant configuration is done below. The "2" in Vagrant.configure
# configures the configuration version (we support older styles for
# backwards compatibility). Please don't change it unless you know what
# you're doing.

ENV["LC_ALL"] = "en_US.UTF-8"

Vagrant.configure("2") do |config|

  config.vm.define "s01" do |host|
    host.vm.box = "ubuntu/xenial64"
    host.vm.hostname = "s01"
    host.vm.network "public_network", ip: "192.168.0.50"
    #host.vm.network "private_network", ip: "10.0.1.1"
    host.vm.provider "virtualbox" do |vb|
      vb.gui = false
      vb.cpus = 4
      vb.memory = "16384"
    end
    host.disksize.size = '128GB'
    host.vm.synced_folder "../../../", "/home/vagrant/st-kilda-pier", owner: "vagrant", group: "vagrant"
  end

end

Multi-VM 예¶

vagrant 초기화

mkdir cluster
cd cluster
vagrant init

설정파일 Vagrantfile의 예, loop를 이용하여 여러개의 VM 과 IP를 설정합니다.

# -*- mode: ruby -*-
# vi: set ft=ruby :

# All Vagrant configuration is done below. The "2" in Vagrant.configure
# configures the configuration version (we support older styles for
# backwards compatibility). Please don't change it unless you know what
# you're doing.

ENV["LC_ALL"] = "en_US.UTF-8"

n_nodes = 5

Vagrant.configure("2") do |config|

  (1..n_nodes).each do |i|
    config.vm.define "c#{'%02d' % i}" do |host|
      host.vm.box = "ubuntu/xenial64"
      host.vm.hostname = "c#{'%02d' % i}"
      host.vm.network "public_network", ip: "192.168.0.5#{i}"
      #host.vm.network "private_network", ip: "10.0.0.#{i}"
      host.vm.provider "virtualbox" do |vb|
        vb.gui = false
        vb.cpus = 4
        vb.memory = "16384"
      end
      host.disksize.size = '64GB'
      host.vm.synced_folder "../../../", "/home/vagrant/st-kilda-pier", owner: "vagrant", group: "vagrant"
    end
  end

end

Vagrant 주요 명령¶

  • vagrant box list
    • box 리스트 출력
  • vagrant box add ubuntu/xenial64
    • box 추가, ubuntu/xenial64(ubuntu 16.04)
    • box 이미지는 https://app.vagrantup.com/boxes/search 에서 검색
  • vagrant box remove ubuntu/xenial64
    • box 삭제
  • vagrant up
    • 가상머신 생성 및 구동
  • vagrant ssh s01
    • s01 hostname 의 가상머신에 접속
  • vagrant halt
    • 가상머신 중지
  • vagrant destroy
    • 가상머신 삭제
